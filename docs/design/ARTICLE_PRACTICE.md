# 文章练习功能设计

## 1. 概述

文章练习是一种更接近真实场景的打字练习模式，用户需要连续输入一段日语文本，模拟实际的文章打字体验。

## 2. 当前问题

- 逐字符输入判定，用户体验不佳
- 不符合文章打字的自然习惯
- 用户需要逐字输入，容易疲劳

## 3. 新设计方案

### 3.1 输入模式：整词/整句输入

**核心机制**：
- 将文章分成若干个"单元"（词语或短句）
- 用户一次输入一个单元的内容
- 输入正确后自动跳到下一个单元

**单元划分规则**：
- 按空格和标点符号划分
- 每个单元长度建议 3-15 个字符
- 特殊情况（如长单词）可单独成单元

### 3.3 输入检测机制

**方案 A：实时匹配（推荐）**
- 用户持续输入，系统实时检测
- 当用户输入与目标单元完全匹配时，自动确认并跳到下一单元
- 无需按回车或空格

**方案 B：空格确认**
- 用户输入完成后按空格确认
- 系统判断正确性
- 正确跳到下一单元，错误提示重试

**方案 C：混合模式**
- 默认使用实时匹配
- 用户也可以按空格/回车手动确认

### 3.4 颜色状态

| 状态 | 颜色 | 说明 |
|------|------|------|
| 未输入 | 灰色 (#94a3b8) | 待输入的单元 |
| 当前 | 靛蓝色 (#6366f1) | 当前需要输入的单元 |
| 已正确 | 绿色 (#22c55e) | 已完成的单元 |
| 错误 | 红色 (#ef4444) | 输入错误 |

### 3.5 统计指标

- **WPM**：每分钟正确输入的字符数
- **正确率**：正确单元数 / 总单元数
- **已正确数**：已成功完成的单元数

## 5. 用户交互流程

1. **进入文章列表** → 选择文章
2. **开始练习** → 显示第一个单元
3. **输入内容** → 实时检测匹配
4. **匹配成功** → 自动跳到下一单元，显示绿色
5. **练习完成** → 跳转到结果页面
6. **查看结果** → 显示 WPM、正确率、统计

## 6. 翻译功能

- 每篇文章可配置全文翻译
- 页面右上角显示"显示翻译"按钮
- 点击展开/收起翻译内容
- 翻译内容存储在 JSON 文件的 `translation` 字段
